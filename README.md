# Nmap Output Parser (Go)

## Overview

This project was created as a small experiment to test the code quality generated by **AI-assisted development**. I was trying **Claude AI** for the first time and decided to give it a practical programming task.

The prompt I gave was simple: **build a command-line Nmap output parser using Go**. The goal was to take the output from an Nmap scan and convert it into a cleaner, more readable format.

After a few iterations (around **3–4 prompt alterations**), the tool started producing the expected results.

This repository contains the result of that experiment.

---

## Purpose

The purpose of this project was to:

* Test how well AI can generate **usable Go code**
* Evaluate the **quality of code produced by AI**
* See how much **prompt iteration** is needed to get a working tool
* Experiment with **automation for parsing Nmap scan results**

---

## Features

* Parses **Nmap scan output**
* Displays scan results in a **clean and structured format**
* Simple **command-line usage**
* Written in **Golang**
* Works well in **Unix pipelines**

Example pipeline usage:

```
sudo nmap -A <target> | ./g-nmap-parser
```

---

## Installation

### 1. Clone the repository

```
git clone --depth=1 https://github.com/sandipduley/G-nmap-parser.git
cd G-nmap-parser
```

### 2. Move to your PATH (optional)

```
sudo cp g-nmap-parser /usr/local/bin/
```

---

## Usage

Run Nmap and pipe the output into the parser:

```
sudo nmap -A <target> | ./g-nmap-parser
```

Example:

```
sudo nmap -A scanme.nmap.org | ./g-nmap-parser
```
